Les deux pages traitent de l'intégration d'outils et de fonctions dans les modèles OpenAI pour enrichir les interactions.

La page "Migrate to the Responses API" cible les développeurs souhaitant migrer vers l'API Responses, une évolution de Chat Completions, offrant une interface unifiée avec plusieurs outils intégrés, meilleure performance et coût réduit. Elle met l'accent sur l'aspect agent intelligent et multimodal.

La page "Function calling" s'adresse aux développeurs voulant connecter les modèles à des données et actions externes via des fonctions définies par schéma JSON ou des outils personnalisés. Elle explique le mécanisme et l'usage des appels de fonctions pour étendre les capacités des modèles, notamment avec les versions GPT-5.4+.

En points clés, la première page présente les bénéfices de l'API Responses et sa simplicité pour créer des agents puissants, tandis que la deuxième détaille la méthode pour exposer des fonctionnalités externes aux modèles via des appels de fonction. La première est une API complète d'agent, la seconde un mécanisme d'extension fonctionnelle.

Limites: Responses API est nouvelle et donc à adopter pour projets neufs, tandis que Function calling nécessite de bien structurer et maintenir les fonctions externes. Les deux approches sont complémentaires dans l'écosystème OpenAI.
